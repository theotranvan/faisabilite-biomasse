# Faisabilité Biomasse - Custom Instructions

## Project Overview

This is a **full-stack Next.js application** for biomass heating feasibility studies, replacing a traditional Excel (.xlsm) file. The app manages multiple projects (affaires) with complex engineering calculations, cost analysis, and financial projections.

### Key Technologies
- **Frontend**: Next.js 14 (App Router) + React 18 + TypeScript + Tailwind CSS
- **Backend**: Next.js API Routes
- **Database**: PostgreSQL + Prisma ORM
- **Auth**: NextAuth.js v4 (to be fully integrated)
- **Charts**: Recharts
- **Export**: jsPDF + html2canvas

## Database Schema

### Core Tables
- **User** : App users with authentication
- **Affaire** : Projects/clients with reference data and parameters
- **Batiment** : Buildings within an affaire (initial & reference states)
- **Parc** : Heating networks (1-4 groupings per affaire)
- **ChiffragReference** : Costing for reference solution
- **ChiffrageBiomasse** : Costing for biomass solution

### Reference Tables
- **Energie** : Energy tariffs (7 types)
- **CaracteristiqueBiomasse** : Fuel properties (4 types)
- **FacteurEmission** : CO2/SO2 emission factors
- **BddCout** : Cost database (~50 items across 6 categories)
- **MeteoMoyenne** : DJU by department (96 departments)
- **MeteoMonotone** : Hourly temperatures (8760/year × 11 towns)
- **PertesReseau** : Network losses by pipe section

## Calculation Modules (CRITICAL - ALREADY IMPLEMENTED)

### chiffrage.ts

### bilan-actualise.ts
- `calculBilan20ans()` - 20-year financial projection
- `calculEmissions()` - CO2/SO2 calculations

- `calculTempsRetourInvestissement()` - Payback period

## Key Design Points

### Calculations
- **All calculations run client-side** using React hooks (useMemo, useCallback)
- **No recalculation API calls** needed - formulas are pure functions
- **Immutable pattern** - calculations return new objects, don't mutate state

### Data Management
- **Prisma for type-safe database access**
- **Normalized schema** prevents data duplication
- **Seed data** ensures consistent reference tables

### UI/UX
- **Tailwind CSS** for responsive design
- **Component-based** architecture
- **Table editors** for bulk data entry
- **Real-time validation** with visual feedback

### Performance
- **Server-side rendering** where possible
- **Incremental Static Regeneration** for reference data
- **Client-side caching** for user preferences

## Common Tasks

### Add a New Calculation
1. Create function in appropriate `src/lib/calculs/*.ts`
2. Write TypeScript interfaces for inputs/outputs
3. Export from `src/types/index.ts`
4. Use in React components with `useMemo()`

### Add a Database Table
1. Define in `src/prisma/schema.prisma`
2. Run `npm run prisma:migrate`
3. Add seed data to `src/prisma/seed.ts` if needed
4. Generate client: `npm run prisma:generate`

### Add an API Route
1. Create file in `src/app/api/[path]/route.ts`
2. Export `GET`, `POST`, `PUT`, `DELETE` as needed
3. Use `db` from `@/lib/db` for database access
4. Return `NextResponse` with data or errors

### Create a Component
1. Create in `src/components/` (organized by feature)
2. Use TypeScript for props
3. Import Tailwind classes
4. Use calculation modules via `useMemo()` if needed

### Database Issues
```bash

# View database

```

### Calculation Errors
- Verify input types match function signatures
- Check for null/undefined values
- Use `console.log()` in React component renders
- Check calculation module tests

- Check headers (X-User-ID for auth)
- Verify request body format
- Use browser DevTools Network tab
- Check server console for errors
